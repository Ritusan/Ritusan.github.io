# Tree Shaking

字面意思是“摇树”，把一个模块里没有引用的东西都摇晃掉，
想实现引入什么，才打包什么，比如引用一个模块中的一个方法，那就只打包这一个方法而不是打包这个模块的所有代码。

可以把一个模块理解成一棵树。

Tree Shaking 只支持ES6 Module，只支持import模块的引入，只支持静态引入的方式，而CommonJS是动态引入的所以不支持。

# 在开发环境中配置Tree Shaking

development的模式默认没有Tree shaking的功能，但可以进行配置。
production模式默认提供了Tree shaking的功能。

module.exports = {
    entry: 'index.js',
    output: {
        path: path.resolve(__dirname, './dist'),
        filename: 'bundle.js'
    },
    plugins: [
        new HtmlWebpackPlugin({
            template: 'src/index.html'
        })
    ],
    // 哪些导出的模块被使用了我们再做打包
    optimization: {
        usedExports: true
    }
}

# Development和Production打包区分

在src同级目录的build文件夹下新建两个文件webpack.dev.js和webpack.prod.js，之后还要建webpack.common.js

webpack.dev.js

const webpack = require('webpack')
const merge = require('webpack-merge')
const commonConfig = require('./webpack.common.js')

const devConfig = {
    mode: 'development',
    devtool: 'cheap-module-eval-source-map',
    devServer: {
        contentBase: './dist',
        open: true,
        port: 8080,
        hot: true
    },
    plugins: [
        new webpack.HotModuleReplacementPlugin()
    ],
    optimization: {
        usedExports: true
    }
}
module.export = merge(commonConfig,devConfig)

因为webpack.dev.js和webpack.prod.js中的代码存在大量的重复，所以可以创建一个webpack.common.js文件

webpack.common.js

const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
const CleanWebpackPlugin = require('clean-webpack-plugin')
module.export = {
    entry: {
        main: './src/index.js',
    },
    plugins: [
        new HtmlWebpackPlugin({
            template: 'src/index.html'
        }),
        new CleanWebpackPlugin(['dist',{
            // 根路径往上走一级
            root: path.resolve(__dirname,'../')
        }])
    ],
    output: {
        filename: '[name].js',
        path: path.resolve(__dirname,'../dist')
    }
}
