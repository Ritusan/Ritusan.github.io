    # Webpack 和 Code Splitting

    Code Splitting：代码分割

    安装lodash：npm install lodash --save,lodash是一个功能集合，提供了很多工具方法

    # 同步加载的代码

    import _ from 'lodash'

console.log(_.join(['a','b','c'],'**'))
    1
    2
    3

    如果把lodash和业务代码一起打包，打包文件会很大，加载时间会长，如果改了业务代码，用户需要重新加载main.js才能获取到最新的代码。

    # 配置webpack自动做代码分割

    // webpack.common.js
module.export = {
    optimization: {
        splitChunks: {
            chunks: 'all'
        }
    }
}

    # 异步加载的代码

    异步载入的组件会自动地打包到一个单独的文件里，不用在配置文件中配置。

    function getComponent() {
    // magicComment:魔法注释
    return import(/*webpackChunkName:"lodash"*/'lodash').then(({ default: _ }) => {
        let ele = document.createElement('div')
        ele.innerHTML = _.join(['a','b'],'-')
        return ele
    })
}

getComponent().then(ele=>{
    document.body.appendChild(ele)
})

    # 总结

    代码分割和webpack无关

    webpack中实现代码分割有两种方式：

    1. 同步代码：只需在webpack.common.js中配置optimization即可
    2. 异步代码：无需做任何配置，会自动进行代码分割，打包放置到新的文件中
